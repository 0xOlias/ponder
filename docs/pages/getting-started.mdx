# Getting started

## A) I'm replacing a Graph Protocol subgraph

Ponder is an alternative to the Graph Protocol. The `create-ponder` CLI tool makes it really easy to migrate a subgraph repository to Ponder.

### 1. Run `create-ponder`

This command will create a project folder called `ponder` in the current working directory. Include the `--from-subgraph` option to bootstrap your project using an existing Graph Protocol subgraph.

<Tabs items={["npm", "pnpm", "yarn"]}>
  <Tab>```bash npm init ponder@latest -- --from-subgraph [path] ```</Tab>
  <Tab>```bash pnpm create ponder --from-subgraph [path] ```</Tab>
  <Tab>```bash yarn create ponder --from-subgraph [path] ```</Tab>
</Tabs>

### 2. Start the development server

<Tabs items={["npm", "pnpm", "yarn"]}>
  <Tab>```bash cd ponder npm run dev ```</Tab>
  <Tab>```bash cd ponder pnpm dev ```</Tab>
  <Tab>```bash cd ponder yarn dev ```</Tab>
</Tabs>

The dev server prints logs to help you debug any configuration issues or errors. The server automatically reloads whenever you save changes in any project file.

### 3. Write event handlers

Ponder event handler functions are similar to Graph Protocol mapping functions.

## B) I'm starting a new Ponder project

### 1. Run `create-ponder`

This command will create a project folder called `ponder` in the current working directory. Include the `--from-etherscan` option with an Etherscan contract link to bootstrap a Ponder project for that contract.

<Tabs items={["npm", "pnpm", "yarn"]}>
  <Tab>```bash npm init ponder@latest -- --from-subgraph [path] ```</Tab>
  <Tab>```bash pnpm create ponder --from-subgraph [path] ```</Tab>
  <Tab>```bash yarn create ponder --from-subgraph [path] ```</Tab>
</Tabs>

### 2. Start the development server

<Tabs items={["npm", "pnpm", "yarn"]}>
  <Tab>```bash cd ponder npm run dev ```</Tab>
  <Tab>```bash cd ponder pnpm dev ```</Tab>
  <Tab>```bash cd ponder yarn dev ```</Tab>
</Tabs>

The dev server prints logs to help you debug any configuration issues or errors. The server automatically reloads whenever you save changes in any project file.

### 3. Paste RPC URL into `.env.local`

Ponder fetches blockchain data from an Ethereum RPC endpoint. You can get an RPC URL from Alchemy, QuickNode, Infura, an Ethereum node on your local machine, etc.

### 4. Write event handler functions

Ponder event handler functions are similar to Graph Protocol mapping functions.
